1. PROJECT OVERVIEW
================================================================================

A full-stack app that lets users create watchlists of Polymarket prediction
markets, track live YES/NO prices, and set threshold rules that flag markets
when the YES price falls within a specified range.

Stack:
  Frontend : React 18 + TypeScript + Vite 5 + Tailwind CSS 3 + axios
  Backend  : ASP.NET Core 8 (C#) Web API + Entity Framework Core 8 (SQL Server)
  Database : SQL Server (LocalDB for dev)
  External : Polymarket Gamma API (search/markets) + CLOB API (midpoints)

4. API ENDPOINTS (BACKEND)
================================================================================

GET    /api/search?q={query}
  → Proxies to Gamma API, returns GammaMarketSummary[]

GET    /api/watchlists
  → Returns WatchlistDto[] (id, name, createdAt)

POST   /api/watchlists
  Body: { "name": string }
  → Returns WatchlistDto

POST   /api/watchlists/{id}/markets
  Body: { "marketId": string, "question": string, "yesTokenId": string, "noTokenId": string }
  → Returns { id, marketId, question, yesTokenId, noTokenId }

DELETE /api/watchlists/{id}/markets/{watchlistMarketId}
  → 204 No Content

GET    /api/watchlists/{id}/view?inRangeOnly=true&sort=yes_desc
  Sort options: "yes_asc", "yes_desc", "flag"
  → Returns WatchlistViewRow[]

POST   /api/rules
  Body: { "watchlistMarketId": int, "lowThreshold": decimal, "highThreshold": decimal }
  Constraints: 0 ≤ low < high ≤ 1
  Note: Replaces any existing rule for that market (MVP: one rule per market)
  → Returns RuleDto

DELETE /api/rules/{id}
  → 204 No Content

6. DATABASE SCHEMA (SQL Server)
================================================================================

TABLE Watchlist
  Id          INT IDENTITY PK
  Name        NVARCHAR(200) NOT NULL
  CreatedAt   DATETIME2 DEFAULT GETUTCDATE()

TABLE WatchlistMarket
  Id              INT IDENTITY PK
  WatchlistId     INT FK → Watchlist(Id) CASCADE
  MarketId        NVARCHAR(100) NOT NULL     -- Polymarket market id/slug
  Question        NVARCHAR(500) NOT NULL
  YesTokenId      NVARCHAR(100) NOT NULL
  NoTokenId       NVARCHAR(100) NOT NULL
  AddedAt         DATETIME2 DEFAULT GETUTCDATE()

TABLE Rule
  Id                  INT IDENTITY PK
  WatchlistMarketId   INT FK → WatchlistMarket(Id) CASCADE
  LowThreshold        DECIMAL(5,4) NOT NULL   -- e.g. 0.6200
  HighThreshold       DECIMAL(5,4) NOT NULL   -- e.g. 0.6800
  CreatedAt           DATETIME2 DEFAULT GETUTCDATE()

TABLE LatestPrice
  TokenId     NVARCHAR(100) PK               -- string primary key
  MidPrice    DECIMAL(10,6) NOT NULL
  UpdatedAt   DATETIME2 DEFAULT GETUTCDATE()

TABLE FlagState
  WatchlistMarketId   INT PK FK → WatchlistMarket(Id) CASCADE
  IsInRange           BIT DEFAULT 0
  CheckedAt           DATETIME2 DEFAULT GETUTCDATE()

Indexes:
  IX_WatchlistMarket_WatchlistId ON WatchlistMarket(WatchlistId)
  IX_Rule_WatchlistMarketId      ON Rule(WatchlistMarketId)

Relationships summary:
  Watchlist 1→N WatchlistMarket
  WatchlistMarket 1→N Rule  (but MVP enforces max 1 rule per market)
  WatchlistMarket 1→1 FlagState
  LatestPrice is standalone, keyed on TokenId (not FK to WatchlistMarket)

7. EF CORE MODEL DETAILS
================================================================================

AppDbContext DbSets:
  Watchlists       → DbSet<Watchlist>
  WatchlistMarkets → DbSet<WatchlistMarket>
  Rules            → DbSet<Rule>
  LatestPrices     → DbSet<LatestPrice>
  FlagStates       → DbSet<FlagState>

Fluent API configuration in OnModelCreating:
  - Table names match SQL exactly: "Watchlist", "WatchlistMarket", "Rule",
    "LatestPrice", "FlagState"
  - LatestPrice: string PK on TokenId, HasMaxLength(100), MidPrice precision(10,6)
  - FlagState: PK = WatchlistMarketId, 1-to-1 with WatchlistMarket via
    HasOne().WithOne().HasForeignKey<FlagState>(x => x.WatchlistMarketId)
  - Rule thresholds: precision(5,4)
  - All cascading deletes from parent tables

Navigation properties:
  Watchlist.Markets          → ICollection<WatchlistMarket>
  WatchlistMarket.Watchlist  → Watchlist
  WatchlistMarket.Rules      → ICollection<Rule>
  WatchlistMarket.FlagState  → FlagState?
  WatchlistMarket.YesPrice   → LatestPrice? (exists in model but NOT mapped in Fluent API)
  WatchlistMarket.NoPrice    → LatestPrice? (exists in model but NOT mapped in Fluent API)
  Rule.WatchlistMarket       → WatchlistMarket
  FlagState.WatchlistMarket  → WatchlistMarket

10. EXTERNAL API DETAILS (Polymarket)
================================================================================

GAMMA API (market search & metadata):
  Base URL: https://gamma-api.polymarket.com
  Search:   GET /public-search?q={query}
    Response: { "events": [ { "markets": [ { ... } ] } ] }
    Market fields used: id, question, slug, clobTokenIds (JSON string array),
                        outcomes (JSON string array like ["YES","NO"])
  Single:   GET /markets/{marketId}

CLOB API (live prices):
  Base URL: https://clob.polymarket.com
  Midpoints: POST /midpoints
    Request body: [{ "token_id": "..." }, ...]
    Response: { "tokenId1": "0.543", "tokenId2": "0.457", ... }
    Values are decimal strings (0 to 1).

Token ID parsing logic (PolymarketService.ParseMarketNode):
  - clobTokenIds is a JSON-encoded string array in the market object
  - outcomes is a JSON-encoded string array like ["YES", "NO"]
  - Match by index: outcomes[i] ↔ clobTokenIds[i]
  - Extract yesTokenId where outcome == "YES", noTokenId where outcome == "NO"

11. BACKGROUND SERVICE BEHAVIOR
================================================================================

PricePollingService (BackgroundService):
  - Interval: 20 seconds
  - Registered via: builder.Services.AddHostedService<PricePollingService>()
  - Creates its own DI scope each tick (IServiceScopeFactory)
  - Tick steps:
    1. Load all WatchlistMarkets (with .Include(m => m.Rules))
    2. Collect all unique YES + NO token IDs
    3. POST to CLOB /midpoints for batch price fetch
    4. Upsert LatestPrice rows (FindAsync → Add or Update)
    5. Evaluate each Rule: YES price in [low, high] → FlagState.IsInRange
    6. Upsert FlagState rows
  - Uses db.SaveChangesAsync twice (once for prices, once for flags)

12. DI REGISTRATION (Program.cs)
================================================================================

  AddDbContext<AppDbContext>(opt => opt.UseSqlServer(...))
  AddHttpClient<PolymarketService>()    ← registers as Transient with named HttpClient
  AddHostedService<PricePollingService>()
  AddControllers()
  AddEndpointsApiExplorer()
  AddSwaggerGen()
  AddCors → allows http://localhost:5173, http://localhost:3000

DO NOT also call AddScoped<PolymarketService>() — that would override
the AddHttpClient registration and break HttpClient injection.

Middleware pipeline:
  app.UseSwagger()
  app.UseSwaggerUI()
  app.UseCors()
  app.MapControllers()
  app.Run()

At startup: db.Database.EnsureCreated() in a scope (dev convenience).

16. KEY BEHAVIORAL NOTES
================================================================================

- Prices are stored as decimals 0.0000–1.0000 (not percentages)
- Frontend multiplies by 100 for display: (v * 100).toFixed(1) + "%"
- Rule thresholds are also stored as decimals (0–1); UI inputs are percentages
  that get divided by 100 before sending to API
- WatchlistDetailPage polls every 30 seconds (POLL_INTERVAL_MS = 30_000)
- Backend polls Polymarket every 20 seconds
- MVP: one rule per market (POST /api/rules replaces existing)
- RulesPage receives question + existing rule via React Router location.state
- Client uses axios with baseURL '/api' (relative, handled by Vite proxy)

18. COMMON PITFALLS FOR AI AGENTS
================================================================================

1. Do NOT register PolymarketService with AddScoped/AddTransient manually.
   It's already registered via AddHttpClient<PolymarketService>().

2. LatestPrice has a STRING primary key (TokenId), not an int.

3. FlagState uses WatchlistMarketId as BOTH its PK and FK — it's a 1-to-1.

4. WatchlistMarket.YesPrice and .NoPrice navigation properties exist in the
   model class but are NOT configured with Fluent API relationships.
   Prices are loaded separately via manual LINQ queries.

5. The Gamma API returns clobTokenIds and outcomes as JSON-encoded STRING
   fields, not as actual JSON arrays. They must be deserialized separately.

6. Rule thresholds use DECIMAL(5,4) — max value 9.9999. Since they represent
   probabilities (0–1), this is fine, but don't use values outside this range.

7. React Router v6 is used — not v5. Use useParams, useNavigate, Routes/Route.

8. Tailwind v3 is used — not v4. Config uses content[] array, not the v4
   @tailwind directives approach.

9. The frontend has NO state management library (no Redux, Zustand, etc.).
   All state is local (useState/useEffect in each page component).

10. There is NO authentication/authorization in the app.

11. The namespace is PolymarketWatchlist (not "Polymarket.Watchlist" etc.).

12. All C# DTOs are records (not classes) in a single Dtos.cs file.

13. EF Core table names are singular: "Watchlist", "Rule", "LatestPrice", etc.
    DbSet property names are plural: Watchlists, Rules, LatestPrices, etc.
